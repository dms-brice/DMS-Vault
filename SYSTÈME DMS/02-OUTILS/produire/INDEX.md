# produire — Ressources, Templates, Arsenal opérationnel

**Parent** : `[[../INDEX]]` (02-OUTILS)

## Objectif

**Boîte à outils vivante**. Tout ce qui est réutilisable en production : templates, modèles, guidelines, ressources.

## Contenu typique

- **templates/** → Modèles de documents (fiches produit, devis, brief)
- **guidelines/** → Tonalité, palette, principes design, voix de marque
- **ressources/** → Images, palettes, références, assets stock
- **processus/** → Checklist, workflows opérationnels

## Convention

- **Template** = prêt à dupliquer & customiser
- **Guideline** = source de truth (version unique)
- **Ressource** = asset réutilisable (image, palette, etc.)

## Important

Quand tu mets à jour une guideline ou un template :
- **DMS** = la met à jour ici
- **Prospects** vault voit automatiquement la nouvelle version (référence DMS)
- **Pas de duplication** = une source de truth

## Workflow

1. Créer / tester template
2. Valider avec Carole (si création)
3. Documenter usage dans ce dossier
4. Prospects vault le référence via `[[DMS Obsidian/02-OUTILS/produire/...]]`
