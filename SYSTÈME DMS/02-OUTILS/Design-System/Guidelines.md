# Guidelines — Bonnes pratiques

**Parent** : `[[INDEX]]`

## Principes universels

### Cohérence visuelle

- Tout asset = palette + typo + composants DMS
- Pas de couleurs "custom"
- Pas de polices externes non-approuvées

### Hiérarchie & lisibilité

- Hiérarchie typographique respectée
- Contraste suffisant (WCAG AA minimum)
- Spacing cohérent (utiliser grille)

### Accessibilité

- Alt text sur images
- Labels explicites sur formulaires
- Couleur ≠ seul moyen de transmettre info
- Focus visible sur éléments interactifs

## Par contexte

### Landing pages (prospection)

- Utiliser composants primaires
- Mettre accent sur appels à action
- Logo visible, lisible
- Respecter layout grid

### Outils (tableaux, formulaires)

- Densité visuelle adaptée à utilité
- Navigation claire, accessible
- États visuels évidents (hover, disabled, etc.)

### Documentation

- Typo lisible en petit format
- Hiérarchie claire
- Images/diagrammes cohérents avec palette

## Checklist avant publication

- [ ] Palette DMS utilisée uniquement
- [ ] Typographie conforme
- [ ] Composants standards ou approuvés
- [ ] Contraste WCAG AA vérifié
- [ ] Responsive (mobile, tablet, desktop)
- [ ] Accessible (alt text, labels, focus)
- [ ] Logo dans contexte approprié
- [ ] Spacing cohérent

## En cas de doute

**Questions fréquentes** :
- "Je peux utiliser [couleur personnalisée] ?" → Non. Utiliser palette ou demander validation.
- "Je dois ajouter [composant nouveau] ?" → Documenter ici d'abord.
- "Quel responsive design ?" → Respecter breakpoints définis.

**Validation** :
- Carole (DA) valide nouvelles variations
- Brice référence ce design system pour prospection
