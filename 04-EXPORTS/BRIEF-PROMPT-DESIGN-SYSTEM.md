# PROMPT — Création du Design System DMS

## CONTEXTE

J'ai créé un système Obsidian + GitHub + Claude pour piloter DMS (Décors des Mers du Sud).

Nous avons clarifié l'identité créative :
- Peintures panoramiques narratives (grands formats)
- Style : réaliste, poétique, historique
- Univers : explorations, botanique, route des épices, mondes lointains
- Intent : raconter une histoire, créer une immersion
- Zuber/indiennage = SOURCE historique + LÉGITIMITÉ (pour pitchs experts Suisse uniquement, pas contenu courant)

Actuellement : Besoin de créer un Design System complet pour DMS qui alimente Claude Design et systématise la création de contenu.

## TÂCHE PRINCIPALE

Créer un Design System Structure pour DMS qui couvre les éléments suivants :

### 1. Palette Couleurs

Basée sur l'univers des "peintures panoramiques narratives"

Livrable :
- Couleurs primaires (inspirées par les décors Carole)
- Couleurs secondaires (compléments)
- Couleurs neutres (pour typographie, backgrounds)
- Codes HEX + usage (titres, corps, accents, etc.)
- Notes WCAG (accessibilité)

### 2. Typographie

Hiérarchie & identité

Livrable :
- Polices : titres, corps, accent
- Poids : regular, medium, bold, extra-bold
- Tailles : h1-h6, body, small, caption
- Line-height & spacing
- Tone-of-voice guide (concret, pas jargon, orienté résultat)

### 3. Logos & Marks

Livrable :
- Version full (logotype + symbol)
- Version icon (symbole seul)
- Version monocolor (noir + blanc)
- Espacements & clear space
- Usage rules (ok/pas ok)

### 4. Univers Visuel

Livrable :
- Style photographique (si photos utilisées)
- Illustrations (style graphique)
- Patterns & textures
- Iconographie (si icônes)
- References visuelles (Zuber historique, botaniques, explorations)

### 5. Composants UI

Livrable :
- Boutons (states: default, hover, active, disabled)
- Cartes (cas clients, projets)
- Formulaires (input, select, textarea)
- Layouts (grid, spacing system)
- Call-to-action elements

### 6. Guidelines & Rules

Livrable :
- Dos & don'ts
- Brand voice examples
- Application examples
- Pre-launch checklist

## CONTRAINTES OBLIGATOIRES

1. Identité = "Peintures panoramiques narratives"
   - Pas de "papier peint" ou "toile" dans le design
   - Visuel doit parler d'immersion, histoire, transformation

2. Deux niveaux de langage
   - Courant : explorations, botanique, mondes lointains, transformation
   - Expert : Zuber, indiennage, manufacture, filiation historique (Suisse/architectes)

3. Ton = Concret, pas flou
   - Pas de "esthétique délicate"
   - Oui à "transformation d'espace", "immersion", "récit"

4. Output = Prêt pour utilisation immédiate
   - Instructions claires pour Carole (la DA)
   - Replicable par Claude pour générer du contenu cohérent
